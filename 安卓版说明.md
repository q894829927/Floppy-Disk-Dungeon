# 《1.44MB：最后的软盘》Android 版

Android 版使用系统原生 Java API 和 Canvas 绘制，不依赖 Unity、Godot、第三方框架或外部资源。地图、像素画面和音效均由代码实时生成。

## 系统要求

- Android 6.0（API 23）或更高版本
- 横屏显示
- 支持触摸屏、键盘或游戏方向键

## 安装

构建后的安装包位于：

```text
dist\FloppyDungeon-Android.apk
```

将 APK 复制到 Android 设备后打开安装。由于使用本地调试证书签名，系统可能要求允许“安装未知应用”。

也可以通过 ADB 安装：

```powershell
& "$env:ANDROID_SDK_ROOT\platform-tools\adb.exe" install -r ".\dist\FloppyDungeon-Android.apk"
```

## 触屏操作

- 屏幕方向键：移动；撞向敌人时进行近战攻击
- `FIRE`：进入火球瞄准状态，再点击一个方向发射
- `MOD`：打开模块管理器
- `HELP`：查看玩法说明
- 升级界面：点击一个模块进行安装
- 模块管理器：点击模块进行安装或删除，点击列表外区域返回

外接键盘仍可使用方向键、`WASD`、`F`、`M`、`H` 和返回键。

## 从源码构建

本机需要安装 Android SDK 和 JDK。项目不依赖 Gradle，构建脚本会直接调用 `javac`、`d8`、`aapt2`、`zipalign` 和 `apksigner`。

在项目目录中运行：

```powershell
.\build-android.ps1
```

脚本会完成以下操作：

1. 编译 Java 源码。
2. 生成 `classes.dex`。
3. 创建并对齐 APK。
4. 使用本地调试证书签名。
5. 验证签名。
6. 检查 APK 是否超过 1.44MB。

首次构建会在 `android` 目录中生成 `debug.keystore`。它仅用于本地测试，不应作为正式商店版本的签名证书。

## 发布提示

如果要发布到应用商店，应当创建并妥善保存正式签名证书，同时提高目标 API 版本以满足商店当时的要求。正式证书丢失后将无法为同一应用发布更新。
