# 《1.44MB：最后的软盘》扩展开发指南

本文档介绍如何在不破坏现有架构和 1.44MB 容量限制的前提下，为游戏增加敌人、能力模块、地图元素、关卡、美术和音效。

## 一、项目结构

```text
Floppy Disk Dungeon/
├─ src/
│  └─ floppy.c              游戏的全部核心代码
├─ dist/
│  └─ FloppyDungeon.exe     构建后的游戏程序
├─ build.ps1                构建及体积检查脚本
├─ README.md                游戏说明
└─ 扩展开发指南.md           本文档
```

游戏采用单文件 C/Win32 架构，地图、美术和音效均由代码生成。绝大多数新内容只需要修改 `src/floppy.c`。

完成修改后，在项目目录中运行：

```powershell
.\build.ps1
```

构建成功后，新程序会生成在：

```text
dist\FloppyDungeon.exe
```

构建脚本会检查最终 EXE 是否超过 1,474,560 字节。

## 二、修改前的建议

1. 每次只增加一种内容，并在修改后立即构建和试玩。
2. 尽量使用枚举或具有明确含义的常量，不要继续增加难以辨认的数字编号。
3. 保持地图尺寸和逻辑分辨率不变，除非准备同时调整整个界面。
4. 新增资源后重新确认发布内容的总容量。
5. 修改存档结构时考虑旧存档兼容性。

## 三、新增敌人

当前敌人使用 `Enemy.type` 区分，类型编号 `0–7` 已被现有敌人占用：

| 编号 | 敌人 |
| --- | --- |
| 0 | 史莱姆 |
| 1 | 骷髅 |
| 2 | 弓箭手 |
| 3 | 蝙蝠 |
| 4 | 炸弹怪 |
| 5 | 守卫 |
| 6 | 巫师 |
| 7 | 首领 |

建议先将编号整理为枚举：

```c
enum EnemyType {
    ENEMY_SLIME,
    ENEMY_SKELETON,
    ENEMY_ARCHER,
    ENEMY_BAT,
    ENEMY_BOMB,
    ENEMY_GUARD,
    ENEMY_WIZARD,
    ENEMY_BOSS,
    ENEMY_MIMIC
};
```

新增敌人通常需要完成以下四步。

### 1. 设置属性

在 `spawnEnemy()` 中设置新敌人的初始生命值和其他状态。

示例：让宝箱怪拥有比普通敌人更多的生命值。

```c
if (type == ENEMY_MIMIC) {
    e->maxhp = e->hp = 6 + floorNo / 2;
}
```

### 2. 编写行动规则

在 `enemyTurn()` 中增加新类型的处理分支。

```c
else if (e->type == ENEMY_MIMIC) {
    if (dist(e->x, e->y, px, py) <= 1) {
        damagePlayer(2, e);
    } else {
        chase(e, 0, 0);
    }
}
```

当前函数的最后一个 `else` 是首领逻辑，因此不能直接让未知类型落入最后分支。增加新敌人时，应当把首领处理改成明确的 `else if (e->type == ENEMY_BOSS)`。

### 3. 设置生成条件

在 `generate()` 中决定新敌人从第几层开始出现，以及出现概率。

推荐根据层数构造敌人池。例如：

```c
int type;

if (floorNo < 3) {
    type = irand(3);
} else if (floorNo < 6) {
    type = irand(7);
} else {
    type = irand(9);
}

spawnEnemy(x, y, type);
```

不要把普通敌人的随机范围扩展到首领编号，否则可能在普通房间中随机生成首领。

### 4. 绘制敌人

在 `drawSprite()` 中为新类型设置颜色和几何造型。

```c
else if (t == ENEMY_MIMIC) {
    rect(d, X + 1, Y + 4, 8, 5, 6);
    rect(d, X + 2, Y + 2, 6, 3, 10);
    rect(d, X + 3, Y + 5, 1, 1, 0);
    rect(d, X + 6, Y + 5, 1, 1, 0);
}
```

尽量继续使用矩形、线条和现有调色板，避免为了一个敌人引入图片解码库。

## 四、新增能力模块

模块数量由以下常量控制：

```c
#define MODS 8
```

模块的名称、说明和容量分别保存在：

```c
static const char *modName[MODS];
static const char *modDesc[MODS];
static const unsigned char modSize[MODS];
```

例如增加一个占用 64KB 的短距离传送模块：

```c
#define MODS 9

static const char *modName[MODS] = {
    /* 原有模块…… */
    "TELEPORT.EXE"
};

static const char *modDesc[MODS] = {
    /* 原有说明…… */
    "Short range teleport"
};

static const unsigned char modSize[MODS] = {
    /* 原有容量…… */
    64
};
```

添加定义后，还要根据模块类型实现实际效果。

### 被动属性模块

在 `recalc()` 中应用最大生命值、攻击力或防御力等属性。

```c
if (hasMod(MOD_ARMOR)) {
    def += 1;
}
```

注意：`recalc()` 可能被反复调用，因此应当先恢复基础属性，再重新计算全部模块效果，避免属性无限叠加。

### 受伤触发模块

在 `damagePlayer()` 中处理护盾、闪避、反伤等能力。

### 击杀触发模块

在 `killEnemy()` 中处理吸血、金币奖励、连杀和掉落等能力。

### 移动触发模块

在 `playerStep()` 中处理冲刺、地雷、隐身或移动后恢复等效果。

### 主动能力模块

为主动能力编写独立函数，并在 `keydown()` 中绑定按键。

```c
static void teleport(void) {
    if (!hasMod(MOD_TELEPORT)) {
        status("TELEPORT.EXE is not installed");
        return;
    }

    /* 检查目标位置并移动玩家 */
}
```

当前模块管理器只支持数字键 `1–8`。模块超过八个后，需要增加翻页功能，例如使用 `PageUp`、`PageDown` 切换页面，每页仍显示八个模块。

`installed` 使用整数位保存安装状态，理论上最多可表示 32 个模块。继续扩展到更多模块时，应改为数组或多个整数。

## 五、新增地图元素

当前地图元素定义为：

```c
enum { WALL, FLOOR, EXIT, CHEST, TRAP };
```

例如新增毒池：

```c
enum { WALL, FLOOR, EXIT, CHEST, TRAP, POISON };
```

然后完成以下三部分。

### 1. 在地图中放置

在 `generate()` 中选择合适的地板位置：

```c
if (floorSpot(&x, &y)) {
    map[y][x] = POISON;
}
```

### 2. 实现触发效果

在 `playerStep()` 中判断玩家是否踩到毒池：

```c
else if (map[py][px] == POISON) {
    damagePlayer(1, 0);
    status("POISON SECTOR: 1 damage");
}
```

如果元素只能触发一次，应当在触发后把地图格恢复为 `FLOOR`。

### 3. 绘制外观

在 `drawMap()` 中增加颜色或简单图案：

```c
else if (map[y][x] == POISON) {
    rect(d, X + 2, Y + 5, 6, 3, 9);
}
```

可用同样方式增加传送门、祭坛、商店、可破坏墙壁和特殊出口。

## 六、新增关卡和首领

游戏目前共有十层，通关判断位于 `nextFloor()`：

```c
if (floorNo > 10) {
    mode = WON;
    return;
}
```

增加层数时，还要同步修改：

1. 右侧信息面板显示的 `/ 10`。
2. 帮助和说明文本中的层数。
3. 首领出现规则。
4. 敌人数量和生命值成长速度。
5. 模块解锁速度。

当前首领每五层出现，首领存活时出口保持锁定：

```c
floorNo % 5 == 0
```

如果增加不同首领，推荐根据具体层数选择类型：

```c
if (floorNo == 5) {
    spawnEnemy(x, y, ENEMY_BOSS_ALPHA);
} else if (floorNo == 10) {
    spawnEnemy(x, y, ENEMY_BOSS_BETA);
}
```

## 七、新增音效

游戏通过以下函数实时合成音效：

```c
sound(频率, 时长毫秒, 是否使用噪声);
```

示例：

```c
sound(900, 60, 0);  /* 高频方波：适合拾取或确认 */
sound(160, 90, 0);  /* 低频方波：适合攻击 */
sound(80, 150, 1);  /* 低频噪声：适合爆炸或受伤 */
```

如果需要更丰富的声音，可以连续播放不同频率，或者扩展 `sound()` 支持三角波、扫频和包络。相比添加 WAV 或 MP3 文件，代码合成几乎不增加发布体积。

## 八、新增美术

游戏内部逻辑分辨率为 320×180，地图格尺寸为 10×10 像素。建议继续使用以下绘图函数：

```c
rect();   /* 填充矩形 */
frame();  /* 矩形边框 */
txt();    /* 文字 */
```

新增视觉内容时应注意：

- 保持像素坐标为整数。
- 使用现有调色板，或谨慎扩充 `col()` 中的颜色。
- 角色应当能在 10×10 像素格内辨认。
- 避免细小文字和抗锯齿效果。
- 优先使用程序绘制，不要引入大型图片解码库。

如果确实需要位图，可以使用很小的单色或索引色数组，并直接编译到 EXE 中。

## 九、新增操作按键

按键处理位于 `keydown()`。

新增快捷键时，应确认没有与现有按键冲突：

| 按键 | 当前功能 |
| --- | --- |
| 方向键 / WASD | 移动和近战攻击 |
| F + 方向键 | 发射火球 |
| M | 模块管理器 |
| H | 帮助 |
| R | 重新开始 |
| Esc | 返回或退出 |

示例：使用 `T` 启动传送模块。

```c
else if (k == 'T') {
    teleport();
}
```

主动能力如果会消耗回合，应在成功执行后调用 `enemyTurn()` 和 `reveal()`。

## 十、存档兼容

当前存档只包含最高层数和已解锁模块数量：

```c
typedef struct {
    int best;
    int unlocked;
} Save;
```

增加模块通常不会破坏旧存档，但旧存档中的 `unlocked` 数值可能导致新模块暂时未解锁。

如果要保存更多数据，建议给存档添加版本号：

```c
typedef struct {
    unsigned int version;
    int best;
    int unlocked;
    int totalWins;
} Save;
```

读取时先检查版本和文件长度。如果格式不匹配，应使用默认值，避免因为旧存档导致游戏异常。

## 十一、控制发布体积

当前构建脚本只检查 `FloppyDungeon.exe` 的大小。只要美术、音效和数据都编译进 EXE，这个检查就是完整的。

如果以后增加外部资源，应改为检查整个发布目录：

```powershell
$total = (Get-ChildItem $out -File | Measure-Object Length -Sum).Sum
$limit = 1440KB

if ($total -gt $limit) {
    throw "发布目录总大小为 $total 字节，超过 1.44MB 限制。"
}
```

推荐的体积控制原则：

1. 优先增加规则和行为，不增加大型资源。
2. 使用代码绘图和代码合成音效。
3. 避免引入 C++ 标准库、图片解码器和音频解码器。
4. 使用短文本和紧凑数据表。
5. 每次构建后记录 EXE 大小变化。
6. 最终发布内容最好控制在 1.2MB 以内，保留足够余量。

## 十二、扩展后的检查流程

每次增加内容后，建议按以下顺序检查：

1. 运行 `.\build.ps1`，确认无编译错误。
2. 确认可执行文件仍小于 1.44MB。
3. 开始新游戏，检查新内容是否能够正常生成。
4. 检查敌人或模块的所有触发条件。
5. 打开模块管理器，检查文字是否超出界面。
6. 测试死亡、重开、返回标题和正常退出。
7. 删除或保留旧存档，分别验证兼容情况。
8. 在另一台普通 Windows 10/11 电脑上进行最终测试。

保持“用规则产生内容”的设计方向，通常可以在只增加几百字节到几KB代码的情况下，为游戏增加大量可玩变化。
